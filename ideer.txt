== Jupyter-notebooks =

== Möjliga saker att analysera: ==
* Vilka språk används?
* Antal (kod)filer/projekt
* I vilken utsträckning används metoder?
* I hur vilken utsträckning används klasser?
* Antal snippets per notebook
* Antal rader kod (per fil, snippet, klass och/eller metod)
* Kod/text-kvot
* Antal utvecklare som har bidragit till filen
* Cyklomatisk komplexitet
* Commit-historik, ålder

* Vilka bibliotek förekommer (tillsammans)?
* Vilka funktioner (från dessa bibliotek) anropas tillsammans? Går det att hitta mönster som exv. skulle kunna slås ihop till egna funktioner?
* Hur används arv? (Används det över huvud taget?)
* Koppling mellan klasser
* Cohesion i klasser
* Hur vanliga är kloner?
* Vilka kloner är mest frekvent förekommande? Varfrån kommer de? (Stackoverflow?)
* Vilka utvecklare ligger bakom de olika snippetsen?
* Hur många utvecklare ligger bakom respektive notebook?
* Antal commits, issues, korrelerat exv. till storlek
* Plottar
* Finns data så att vi kan köra?


== Frågor om kloner ==

Hur många kloner per notebook
Hur många kloner per notebookstorleksintervall (mätt i LOC resp bytes)
Hur många notebooks per klon
Klonernas storleksfördelning (i LOC)
Antalet andra notebooks som varje notebook delar snippets med
Antalet Klonkällor per notebooks
Kloner per förekomst
Kloner per språk / storlek (LOC, #snippets) / ...

Kvalitativ analys av X vanligaste klonerna
  Vad innehåller de?
  Samband?
Kvalitativ analys av X minsta/största/median-klonerna
  Vad innehåller de?

Finns det "helt" klonade notebooks? Hur många?
Hur många notebooks har  bara unika snippets? Inga snippets?
Notebooks per klon-%-intervall (0-5, 5-10, etc.)
Sannolikhet att flera kloner finns från samma källa

Är notebooks från samma repo mer lika? 

Om vi har metadata från GitHub om repos:
  "korrelation" olika klonmått mot repots ålder/aktivitet/#committs

Hur stor andel av snippetsen är kloner respektive unika? Totalt? I en genomsnittlig fil?
